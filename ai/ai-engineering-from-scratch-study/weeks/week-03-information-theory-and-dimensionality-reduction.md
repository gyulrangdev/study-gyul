## 핵심 개념 1: 정보이론 (Information Theory)

> 한 줄 요약: 정보이론은 "놀람(surprise)"을 수치화한다. 우리가 매일 쓰는 손실 함수가 여기서 나온다.

### 정보량과 엔트로피

- **정보량(surprise)**: `I(x) = -log(p(x))`. 드문 사건일수록 정보량이 크다. 확실한 사건(p=1)은 정보량 0.
- **엔트로피**: 분포 전체에 걸친 평균 놀람. `H(P) = -Σ p(x) log p(x)`.
  - 공정한 동전: 1 bit (최대). 편향된 동전(99:1): 0.08 bit.
  - 엔트로피는 "압축의 하한선" — 이보다 더 줄일 수 없는 불확실성.
- **단위**: log 밑이 2면 bit, e면 nat. PyTorch/TensorFlow는 기본 nat. `1 nat ≈ 1.4427 bit`.

### 크로스 엔트로피 (우리가 매일 쓰는 손실)

- `H(P, Q) = -Σ p(x) log q(x)`. 진짜 분포 P를 모델 분포 Q로 인코딩할 때의 평균 놀람.
- 분류에서 P는 원-핫 → **`H(P,Q) = -log q(정답클래스)`** 로 단순화. 이게 곧 크로스 엔트로피 손실.
- Q가 P와 완벽히 같으면 크로스 엔트로피 = 엔트로피. 어긋날수록 커진다.

### KL 발산

- `D_KL(P‖Q) = Σ p(x) log(p(x)/q(x)) = H(P,Q) - H(P)`.
- 즉 **크로스 엔트로피 = 엔트로피 + KL 발산**. 학습 중 H(P)는 상수이므로, 크로스 엔트로피 최소화 = KL 발산 최소화 = 모델 분포를 진짜 분포로 밀어붙이기.
- 비대칭: `D_KL(P‖Q) ≠ D_KL(Q‖P)`. 진짜 거리(metric) 아님.

### 크로스 엔트로피가 "그" 분류 손실인 이유 (3가지 관점, 같은 결론)

1. **정보이론**: 진짜 분포 대신 모델 분포를 써서 낭비하는 bit 수. 최소화 = 가장 효율적인 인코더.
2. **최대우도**: `-Σ log q(y_i)` = 음의 로그우도(NLL). 크로스 엔트로피 최소화 = 학습 데이터 우도 최대화.
3. **그래디언트**: logit에 대한 크로스 엔트로피의 그래디언트가 `(예측 - 정답)`. 깔끔·안정 → softmax와 완벽한 짝.

### 상호정보량 / 조건부·결합 엔트로피

- `I(X;Y) = H(X) - H(X|Y) = H(Y) - H(Y|X) = H(X)+H(Y)-H(X,Y)`.
- 독립이면 0, 같은 변수면 `I(X;X)=H(X)`. 대칭적(KL과 달리).
- **특징 선택**: 상관계수는 선형만 잡지만 상호정보량은 어떤 종류의 의존성이든 잡는다 → 특징 순위 매기기에 좋음.
- 결정 트리: 각 분할에서 `H(Y|X)`를 가장 많이 줄이는 특징을 고른다.

### 라벨 스무딩 · 퍼플렉서티

- **라벨 스무딩**: `soft = (1-ε)·hard + ε/K`. 타깃 엔트로피를 올려 과신 방지, 정칙화, 캘리브레이션 개선.
- **퍼플렉서티**: 크로스 엔트로피의 지수 `PP = e^{H(P,Q)}`. "모델이 평균적으로 몇 개 선택지 사이에서 헷갈리는가". 낮을수록 좋음. GPT-2 ≈ 30.

## 핵심 개념 2: 차원 축소 (Dimensionality Reduction)

> 한 줄 요약: 고차원 데이터에도 구조가 있다. 올바른 각도에서 보면 그 구조가 드러난다.

### 차원의 저주

- **거리가 무의미해짐**: 차원이 커지면 임의의 두 점 사이 거리가 모두 비슷해져 최근접 이웃 탐색이 망가진다.
- **부피가 모서리로 몰림**: d차원 초입방체는 `2^d`개 모서리. 데이터가 가장자리로 흩어짐.
- **필요 데이터가 지수적으로 증가**: 2D→20D면 같은 밀도 유지에 `10^18`배 데이터 필요.

### PCA — 분산이 큰 방향 찾기

알고리즘:
1. 중심화 (각 특징에서 평균 빼기)
2. 공분산 행렬 계산
3. 고유분해 (주성분 방향 찾기)
4. 고유값 기준 정렬 (분산 큰 순)
5. 투영 (상위 k개 고유벡터만 남기고 나머지 버림)

- 공분산 행렬은 대칭·양의 준정부호 → 고유벡터가 직교, 고유값이 각 방향의 분산 크기.
- **설명분산비** `= 고유값_k / Σ고유값`. 누적이 0.90~0.95 되는 지점까지 성분을 남긴다(엘보 방법).
- **복원 오차** = 버린 고유값의 합. PCA-기반 이상탐지에도 활용(복원 오차 큰 샘플 = 이상치).

### t-SNE / UMAP — 이웃 관계 보존 (시각화용)

- **t-SNE**: 고차원 이웃 확률분포를 2D에서도 유지. 비선형, 확률적(실행마다 다름), perplexity(5~50)로 이웃 수 조절. **클러스터 간 거리는 의미 없음**, 느림(O(n²)).
- **UMAP**: t-SNE보다 빠르고 전역 구조를 더 잘 보존. `n_neighbors`(지역/전역), `min_dist`(밀집도).
- **커널 PCA**: 커널 트릭으로 비선형 매니폴드 분리 (동심원 데이터 → RBF 커널로 선형 분리 가능).

### 언제 무엇을 쓸까

| 방법 | 용도 | 보존 대상 | 속도 |
| --- | --- | --- | --- |
| PCA | 전처리 / 데이터 압축 | 전역 분산 (선형) | 빠름 (수백만 샘플 OK) |
| t-SNE | 논문급 2D 시각화 | 지역 이웃 | 느림 (<10k 권장) |
| UMAP | 대규모 2D 시각화 | 지역 + 일부 전역 | 중간 |

- 규칙: **전처리·압축엔 PCA, 2D 구조 시각화엔 t-SNE/UMAP.**
