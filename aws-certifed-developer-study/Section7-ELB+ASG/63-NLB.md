# 주요 내용 요약

네트워크 로드 밸런서(NLB)의 특징과 사용 사례

# NLB 기본 특징

## 프로토콜 지원

- Layer 4 로드 밸런서 (TCP/UDP)
- TCP와 UDP 트래픽 처리 가능

## 성능

- 초당 수백만 요청 처리 가능
- 지연 시간: ~100ms (ALB: ~400ms)
- 고성능 처리 가능

## IP 주소 관리

- 가용 영역당 하나의 고정 IP 제공
- 탄력적 IP 할당 가능
- 고정 IP가 필요한 애플리케이션에 적합

# NLB 대상 그룹 옵션

1. EC2 인스턴스

   - TCP/UDP 트래픽 직접 전달

2. IP 주소

   - 프라이빗 IP 주소만 가능
   - 온프레미스 데이터센터 서버 연결 가능

3. Application Load Balancer
   - NLB + ALB 조합 구성 가능
   - 고정 IP (NLB) + HTTP 규칙 (ALB) 활용

# 상태 확인 프로토콜

- TCP
- HTTP
- HTTPS

# 시험 관련 중요 포인트

1. NLB 사용이 적합한 상황:

   - TCP/UDP 트래픽 처리 필요
   - 고성능/낮은 지연시간 요구
   - 고정 IP 주소 필요
   - 초당 수백만 요청 처리 필요

2. 프리티어 미포함

3. 대상 그룹 구성 옵션 이해

   - EC2, IP 주소, ALB 조합 가능

4. 상태 확인 프로토콜 지원 범위
