# 주요 내용 요약

로드 밸런서의 고급 네트워크 보안 설정과 리스너 규칙 구성 방법

# 네트워크 보안 구성

## EC2 인스턴스 보안 그룹 설정

- 기존: 모든 트래픽 허용
- 권장: 로드 밸런서를 통한 트래픽만 허용
- 설정 방법:
  - HTTP 규칙 수정
  - 소스를 로드 밸런서의 보안 그룹으로 지정
  - CIDR 대신 보안 그룹 참조 사용

# ALB 리스너 규칙

## 규칙 구성 요소

1. 조건(Conditions)

   - 호스트 헤더 (\*.example.com)
   - 경로 (Path)
   - HTTP 요청 메소드 (GET/POST)
   - 소스 IP
   - 쿼리 문자열
   - HTTP 헤더

2. 액션(Actions)
   - 대상 그룹으로 전달
   - URL 리다이렉션
   - 고정 응답 반환

## 규칙 우선순위

- 1~50,000 범위 설정 가능
- 낮은 숫자가 높은 우선순위
- 여러 조건 충족 시 우선순위가 높은 규칙 적용

# 실습 예제

## 에러 페이지 규칙 설정

```
조건:
- Path: /error

액션:
- 고정 응답
- HTTP 404
- 메시지: "Not found, custom error"

우선순위: 5
```

# 시험 관련 중요 포인트

- 보안 그룹 간 참조 관계 이해
- ALB 리스너 규칙의 구성 요소
- 규칙 우선순위 체계
- 다양한 조건과 액션 타입
