# 주요 내용 요약

네트워크 로드 밸런서(NLB) 실습 구성 과정

# NLB 생성 단계

## 1. 기본 설정

- 이름: DemoNLB
- 체계: Internet-facing
- IP 유형: IPv4
- 네트워크 매핑: 모든 가용영역 선택

## 2. IP 주소 설정

- AWS 관리 IP 주소 또는 Elastic IP 선택 가능
- 각 가용영역마다 고정 IP 할당

## 3. 보안 그룹 설정

```
보안 그룹 이름: demo-sg-nlb
인바운드 규칙:
- 프로토콜: HTTP (80)
- 소스: 0.0.0.0/0
```

## 4. 리스너 및 라우팅

- 프로토콜: TCP
- 포트: 80
- 대상 그룹 설정

## 5. 대상 그룹 설정

```
이름: demo-tg-nlb
프로토콜: TCP
포트: 80
상태 확인:
- 프로토콜: HTTP
- 임계값: 2
- 타임아웃: 2초
- 간격: 5초
```

# 트러블슈팅

## EC2 보안 그룹 설정

```
인바운드 규칙:
- HTTP: ALB 보안 그룹 허용
- HTTP: NLB 보안 그룹 허용
```

# 주요 설정 포인트

1. 각 가용영역별 IP 주소 관리
2. 보안 그룹 설정
3. 상태 확인 프로토콜 선택
4. EC2 인스턴스의 보안 그룹 규칙 추가

# 시험 관련 중요 포인트

1. NLB와 EC2 간 통신을 위한 보안 그룹 설정
2. 상태 확인 프로토콜 옵션 (TCP, HTTP, HTTPS)
3. IP 주소 할당 방식 (AWS 관리 또는 Elastic IP)
4. 리스너 프로토콜 옵션 (TCP, UDP, TLS, TCP_UDP)
