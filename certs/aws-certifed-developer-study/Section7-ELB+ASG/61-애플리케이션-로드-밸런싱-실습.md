# 주요 내용 요약

애플리케이션 로드 밸런서(ALB) 실습을 통한 로드 밸런싱 구현 방법

# 실습 과정

## EC2 인스턴스 설정

- 두 개의 EC2 인스턴스 생성
  - Amazon Linux AMI 사용
  - t2.micro 인스턴스 타입
  - 보안 그룹: HTTP 및 SSH 트래픽 허용
  - EC2 사용자 데이터로 웹 서버 설정

## 로드 밸런서 유형

- Application Load Balancer (ALB): HTTP/HTTPS 트래픽용
- Network Load Balancer (NLB): TCP/UDP 트래픽, 고성능 필요시
- Gateway Load Balancer (GWLB): 보안, 침입 감지, 방화벽용

## ALB 구성 단계

1. 로드 밸런서 생성

   - 이름: DemoALB
   - 체계: 인터넷 연결형
   - IPv4 사용
   - 모든 가용영역 배포

2. 보안 그룹 설정

   - 새 보안 그룹 생성 (demo-sg-load-balancer)
   - HTTP 트래픽만 허용

3. 대상 그룹 설정
   - 이름: demo-tg-alb
   - 프로토콜: HTTP (포트 80)
   - 두 EC2 인스턴스를 대상으로 등록

# 주요 기능

- 트래픽 분산

  - 두 EC2 인스턴스 간 트래픽 균등 분배
  - 페이지 새로고침 시 다른 인스턴스로 요청 전달

- 헬스 체크
  - 인스턴스 상태 모니터링
  - 비정상 인스턴스 자동 감지
  - 정상 인스턴스로만 트래픽 전달

# 시험 관련 중요 포인트

- 로드 밸런서 유형별 차이점 이해
  - ALB: Layer 7 (HTTP/HTTPS)
  - NLB: Layer 4 (TCP/UDP)
  - GWLB: 보안 및 네트워크 분석용
