# Day 5 — 프롬프트 기초 + 토큰·비용

**목표**: 프롬프트 구조와 토큰·비용 관계를 이해한다.

**개념** — 입력+출력 토큰 모두 과금. 긴 컨텍스트(RAG)는 비용·지연↑. 비용 절감 = 프롬프트 캐싱·배치 추론. 구조화 프롬프트(역할·제약·예시·출력형식)가 일관성↑.

**핸즈온** — Day 3 코드에 system 프롬프트 추가 + 출력을 JSON으로 강제.

**체크**

- [ ] 입력·출력 토큰 모두 과금됨을 안다
- [ ] 비용 절감 수단 2가지를 안다
- [ ] 출력 형식 강제 방법을 안다
